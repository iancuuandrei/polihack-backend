# Ingestion Canonical Bundle

Handoff 06 exports a deterministic, file-based canonical bundle that can be
loaded locally by Handoff 03 and imported later by Handoff 04. The bundle is
import-ready when `validation_report.import_blocking_passed` is `true`.

## Artifacts

- `legal_units.json`: canonical LegalUnit records. `LegalUnit.raw_text` is the
  only citable legal text.
- `legal_edges.json`: deterministic graph edges. In V1 this is limited to
  structural `contains` edges.
- `reference_candidates.json`: unresolved or candidate-only references extracted
  from legal text. These are not legal edges.
- `legal_chunks.json`: retrieval chunks derived from LegalUnit. In V1 each chunk
  is 1:1 with one LegalUnit.
- `embeddings_input.jsonl`: deterministic JSONL input for a future embedding
  job. No vectors are generated by Handoff 06.
- `corpus_manifest.json`: bundle metadata, counts, hashes, input files, and
  local fixture warnings.
- `validation_report.json`: quality metrics, warnings, blocking errors, and
  demo/import readiness gates.

## Handoff 03 Consumption

Handoff 03 can consume:

- `legal_units.json` for `RetrievalCandidate.unit` and `EvidenceUnit`.
- `legal_edges.json` for graph expansion and parent/child context.
- `legal_chunks.json` for lexical/vector retrieval input when retrieval adapters
  need `retrieval_text`.

`LegalChunk.retrieval_text` is derived from deterministic context plus
`LegalUnit.raw_text`. It is useful for scoring and embeddings, but it is not
citable as law. Evidence display must use `LegalUnit.raw_text`.

## Handoff 04 Consumption

Handoff 04 can consume:

- `legal_units.json` for `legal_units` import.
- `legal_edges.json` for graph import.
- `corpus_manifest.json` and `validation_report.json` for import gates and
  provenance.
- `embeddings_input.jsonl` later, after a real embedding job generates vectors.

Handoff 06 does not perform DB import, pgvector import, or embedding generation.

## V1 Policies

- Unknown stays unknown: missing `source_url`, `source_id`, dates, status, and
  resolved references remain null, `"unknown"`, or empty arrays.
- `reference_candidates.json` is not promoted into `legal_edges.json`.
- Unresolved references are non-blocking for local fixtures, but they are
  reported in validation warnings and metrics.
- Local fixtures may have `source_url_coverage = 0.0`; this is allowed only when
  the manifest and validation warnings explain the local demo source.

## Local Helpers

- `ingestion.bundle_loader.load_canonical_bundle(path)` loads all artifacts from
  a standard bundle directory or from the prefixed Codul Muncii fixture files.
- `ingestion.local_retriever.LocalBundleRetriever` performs deterministic local
  lexical scoring over `legal_chunks.retrieval_text` without DB, network,
  vectors, or LLMs.
